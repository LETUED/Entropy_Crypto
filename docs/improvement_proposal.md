## 1. 가설 분해 (Hypothesis Decomposition)

### 문제
현재 문서는 동시에 여러 가설을 주장한다. 하나라도 틀리면 전체가 무너지므로, 의존성 순서대로 분리해야 한다.

### 분해된 하위 가설

| 번호 | 가설 | 의존 관계 |
|---|---|---|
| **H1** | 엔트로피가 낮은 구간에서 ML 예측 정확도가 향상된다 | 독립 (선행 논문 검증됨 ✅) |
| **H2** | 엔트로피 하위 5% 구간에서 이후 변동폭이 평균의 2.89배다 | 독립 — **먼저 검증할 가설** |
| **H3** | 엔트로피 낮은 구간에서 RSI 방향 신호 정확도가 다른 구간보다 높다 | H2 검증 후 |
| **H4** | 엔트로피 필터 + RSI 조합이 RSI 단독보다 수익성이 높다 | H3 검증 후 |

### 권장 검증 순서
```
H2 검증 → H3 검증 → H4 백테스팅
(H2가 틀리면 H3, H4 진행 불필요)
```

## 2. 방향 예측의 논리 모순 해결

### 문제
초기 연구에서 스스로 적은 내용:
> 수익의 87.8%가 "언제 변동이 크냐"를 아는 것에서, 방향 예측 기여 0.0%

그러면서 시스템은 RSI로 방향을 판단한다. 수학적으로 일관성이 없다.

### 선택지

**A안 — 방향 포기, 순수 변동성 전략**
- 엔트로피 낮은 구간 감지 → 롱-숏 동시 포지션 (스트래들)
- 방향 판단 없이 변동폭만 먹음
- 수학적으로 더 일관성 있음
- 코인장: 무기한 선물 양방향 포지션으로 구현 가능

**B안 — RSI를 더 강한 방향 신호로 교체**
- RSI 대신 Order Flow Imbalance, Funding Rate, 고래 순매수 등 사용
- 단순 RSI는 알파가 거의 소진된 지표이므로 방향 신호를 강화해야 함

**권장: A안으로 먼저 시작**, B안은 A안 검증 후 비교

## 3. 엔트로피 임계값 결정 방법

### 문제
현재 코드 예시:
```python
def is_predictable_zone(H_total, threshold=0.3):
    return H_total < threshold
```
`0.3`의 근거가 없다. 테스트 데이터를 보고 결정하면 룩어헤드 바이어스다.

### 개선 방법
```
1. 데이터를 훈련(60%) / 검증(20%) / 테스트(20%)로 3분할
2. 훈련 데이터에서만 엔트로피 분포를 계산
3. 훈련 데이터의 하위 5th / 10th / 20th percentile을 후보 임계값으로 설정
4. 검증 데이터에서 최적 임계값 선택
5. 테스트 데이터는 최종 한 번만 사용
```

임계값은 절대값(0.3)이 아닌 **분위수(percentile)** 기반으로 정의해야 데이터셋 간 이식성이 생긴다.

## 4. 순열 엔트로피 파라미터 고정

### 문제
순열 엔트로피의 두 파라미터가 결과를 크게 바꾼다:

```python
# 파라미터가 결정되지 않음
PE = permutation_entropy(series, m=?, tau=?)
```

멀티스케일까지 더하면 파라미터가 수십 개다. 데이터를 보고 선택하면 과적합이다.

### 개선 방법
선행 논문(GARCH 대비 34% 우수 논문: preprints.org/manuscript/202511.1980)에서 검증된 파라미터를 그대로 사용한다. 직접 최적화하지 않는다.

**권장 시작값 (문헌 기반):**
```python
m = 3          # embedding dimension
tau = 1        # time lag
scales = [1, 2, 4, 8]   # 멀티스케일 타임스케일 (시간봉 기준)
```
파라미터 변경은 H2 가설이 검증된 이후에만 실험한다.

## 5. 백테스팅 방법론 명세

### 현재 상태
로드맵에 "단계 6: 백테스팅"만 있고, 구체적인 방법이 없다.

### 필수 요소

| 항목 | 명세 |
|---|---|
| **검증 방식** | Walk-forward validation (슬라이딩 윈도우) |
| **수수료** | 바이낸스 Maker 0.02% / Taker 0.05% 양방향 포함 |
| **슬리피지** | 시장가 주문 가정, 0.05% 추가 |
| **데이터 기간** | 훈련 2021-2023 / 검증 2024 / 테스트 2025 |
| **기준선(Baseline)** | BTC 바이앤홀드 / RSI 단독 / 랜덤 전략 |
| **평가 지표** | Sharpe Ratio, Max Drawdown, Win Rate, Profit Factor |

### Walk-forward 구조
```
[훈련 1년] → [테스트 3개월] → 슬라이드
             [훈련 1년] → [테스트 3개월] → 슬라이드
                          ...
```
단순 인샘플 백테스팅은 하지 않는다.

## 6. 대안 가설 배제 실험

### 문제
"엔트로피 낮은 구간 = 변동폭 큼"이 사실이더라도, 엔트로피 고유의 효과인지 확인해야 한다.

### 경쟁 지표 비교 실험

다음 지표들로 동일한 실험을 수행해서 엔트로피의 추가적 설명력을 입증해야 한다:

| 경쟁 지표 | 측정하는 것 | 비교 이유 |
|---|---|---|
| 볼린저 밴드 수축 (BBW) | 변동성 압축 | 엔트로피와 유사한 효과 가능성 |
| ATR 저점 | 절대 변동성 낮음 | 단순 변동성 감소와 구분 |
| 거래량 이동평균 | 거래량 감소 | 유동성 감소 효과와 구분 |
| VIX (주식장) | 내재 변동성 | 옵션 시장의 기대치와 비교 |

**목표:** 엔트로피 필터가 이 지표들보다 통계적으로 유의미하게 더 나은 신호를 제공함을 보여준다.

## 7. 통계 검정 명세

### 현재 상태
"엔트로피 최하위 5% 구간에서 2.89배 변동폭" — p-value 없음.

### 필수 통계 검정

**H2 검증:**
```python
from scipy.stats import mannwhitneyu

# 엔트로피 하위 5% 구간의 이후 변동폭
low_entropy_returns = ...
# 나머지 95% 구간의 이후 변동폭
other_returns = ...

stat, p_value = mannwhitneyu(low_entropy_returns, other_returns, 
                              alternative='greater')
# p < 0.05 이어야 가설 채택
```

**다중 검정 보정 (Multiple Testing Correction):**
여러 파라미터 조합을 실험하면 우연히 유의미한 결과가 나올 확률이 높아진다. Bonferroni 보정 또는 FDR(False Discovery Rate) 제어를 적용한다.

## 8. 전체 개선 로드맵

```
Phase 1 — 핵심 가설 단독 검증 (H2)
├── BTC 1H 캔들 데이터 수집 (2021~2025)
├── 멀티스케일 순열 엔트로피 계산 (파라미터 고정)
├── 엔트로피 하위 5% 구간 식별
├── 이후 N시간 변동폭 측정
├── Mann-Whitney U test
└── 경쟁 지표(BBW, ATR)와 비교

Phase 2 — 방향 신호 검증 (H3)
├── A안: 순수 변동성 전략 구현 및 백테스팅
├── B안: RSI 방향 정확도 구간별 비교
└── A안 vs B안 성능 비교

Phase 3 — 통합 백테스팅 (H4)
├── Walk-forward validation
├── 수수료 + 슬리피지 포함
├── Baseline 3종과 비교
└── 통계적 유의성 확인

Phase 4 — 복잡도 증가
├── H_onchain 추가 (온체인 데이터)
├── H_news 추가 (뉴스 감성)
└── En-BMA 통합
    → Phase 3가 성공한 경우에만 진행
```

## 핵심 원칙 요약

1. **H2 먼저** — 나머지 모든 것은 H2 위에 쌓인다
2. **파라미터를 데이터 전에 고정** — 과적합 방지
3. **방향 전략은 선택 후 집중** — A안(변동성) vs B안(방향) 동시 추진 금지
4. **경쟁 지표와 비교** — "엔트로피가 필요하다"는 것을 직접 보여야 한다
5. **수수료 포함** — 수수료 없는 백테스팅 결과는 의미 없다
